## ドメイン用語

| 用語 | 英語 / コード上の名称 | 定義 |
|---|---|---|
| 物件概要書 | document | 不動産物件の基本情報が記載された書類（画像/PDF）。OCR処理の入力となる。 |
| アップロード | upload | ユーザーが物件概要書ファイルをシステムに送信すること。`POST /api/documents` で受け付ける。 |
| 抽出データ | extraction | OCRエンジンが物件概要書から読み取った生テキストと座標情報（bounding_boxes）。`extractions` テーブルに保存する。加工・集約しない。 |
| 正規化データ | normalized_property | LLMがOCR生テキストから構造化した、編集可能な物件情報。`normalized_properties` テーブルに保存する。 |
| リビジョン | revision | ユーザーが正規化データを編集した際の差分（before/after）。`revisions` テーブルに追記される。 |
| ジョブ | job | 非同期処理の単位。文書1件のアップロードに対して1つ作成される。`queued → processing → succeeded/failed` の状態遷移を持つ。 |
| ジョブキュー | job queue | `jobs` テーブルの `status = queued` のレコード群。WorkerがSQLでポーリングして取得する。 |
| OCRプロバイダー | OCRProvider | OCR処理を行う外部サービスの抽象インターフェース。Phase 0 の実装は Google Vision API。 |
| エクストラクター | Extractor | LLMを用いてOCR生テキストから構造化フィールドを抽出するコンポーネントの抽象インターフェース。Phase 0 の実装は OpenAI Structured Outputs。 |
| ストレージアダプター | StorageAdapter | ファイルのアップロード・署名URL生成・削除を行う抽象インターフェース。Phase 0 の実装は S3StorageAdapter。 |
| 署名URL | signed URL | Storageに保存されたファイルへの期限付きアクセスURL。OCRプロバイダーへの画像受け渡しに使用する。 |
| Worker | worker | Cloud Run 上で動作する非同期処理プロセス。ジョブキューをポーリングし、OCR・LLM・DB書き込みを実行する。APIレイヤーでは実行しない。 |

## 技術用語（設計判断に関わるもの）

| 用語 | 定義 |
|---|---|
| Adapterパターン | OCR / LLM / Storage を抽象インターフェース経由で利用し、実装を差し替え可能にする設計。アプリ層がSDKを直接参照することを禁止する。 |
| 同期処理禁止原則 | APIレイヤーでOCR・LLMを同期実行することを禁止するルール。全ての重処理は必ずWorkerが担う。 |
| データ分離原則 | `extractions`（生OCR）と `normalized_properties`（確定済み）を別テーブルに保持し、統合・マージしないルール。 |
| extractor_version | LLMの抽出スキーマのバージョン。スキーマ変更時に後方互換を管理するために `extractions` テーブルに記録する。 |
| x-user-id | Phase 0 における暫定認証ヘッダー。NextAuth 統合までの代替手段。値は UUID 形式の users.id。 |

## 状態・ステータス

| 値 | 意味 |
|---|---|
| `queued` | ジョブがWorker待ち状態 |
| `processing` | WorkerがOCR/LLM処理を実行中 |
| `succeeded` | 処理が正常完了し、normalized_properties に保存済み |
| `failed` | エラーが発生し、retry上限（3回）を超過した |
